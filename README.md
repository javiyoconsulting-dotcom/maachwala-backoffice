# Backoffice Microservice

Node.js microservice exposing:

```text
POST /backoffice/onboardorg
```

The service includes CORS headers and supports browser preflight requests with `OPTIONS`.

## Run

```bash
npm install
npm start
```

The service starts on port `3000` by default. To use another port:

```bash
$env:PORT=4000; npm start
```

## Cloud Run

The service includes a `Dockerfile` and GitHub Actions workflow for Cloud Run. Configure these GitHub repository secrets before running `.github/workflows/deploy-cloud-run.yml`:

```text
GCP_PROJECT_ID
GCP_REGION
GCP_WORKLOAD_IDENTITY_PROVIDER
GCP_SERVICE_ACCOUNT
DATABASE_URL
```

The workflow deploys the service as `backoffice-microservice` and sets `PGSSL=true` automatically.

## PostgreSQL

Set the PostgreSQL pooler connection string in `DATABASE_URL`. Keep the real password in your local environment, not in source code.

```bash
$env:DATABASE_URL="postgresql://postgres.ofqhwumptcehlzialsjg:PASSWORD@aws-1-ap-south-1.pooler.supabase.com:6543/postgres"
$env:PGSSL="true"
npm start
```

Because the provided password contains `@`, encode that character as `%40` when placing it inside `DATABASE_URL`.

Before running the service, create the target schema/table:

```bash
npm run migrate
```

The service writes onboarding records to `core.contractedorg`. The incoming payload maps to `name`, `usercount` from `headcount`, `businessregion`, `businesstype`, and `data`. The database field `"number"` is generated by the service using the current Unix timestamp in milliseconds, for example `1783123456789`. The database field `status` is always saved as `true`.

## Request

```bash
curl -X POST http://localhost:3000/backoffice/onboardorg ^
  -H "Content-Type: application/json" ^
  -d "{\"name\":\"demo\",\"headcount\":4,\"status\":\"ACTIVE\",\"data\":{\"owner\":\"name\",\"ownerphone\":9875352512}}"
```

## Input JSON

```json
{
  "name": "demo",
  "headcount": 4,
  "businessregion": 1,
  "businesstype": 2,
  "data": {
    "owner": "name",
    "ownerphone": 9875352512
  }
}
```
